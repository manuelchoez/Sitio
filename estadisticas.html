Resultados

Una vez que ha agregado un equipo a Observium y ha comenzado a recopilar datos, puede interpretar los resultados para obtener información valiosa sobre el rendimiento y la disponibilidad del equipo. Aquí hay algunos de los tipos de información que puede obtener de los resultados de Observium:

    1. Estado actual: Observium proporcionará un estado general del equipo, que puede ser "Up" (en línea) o "Down" (fuera de línea).
    2. Gráficos de rendimiento: Observium generará gráficos para mostrar el rendimiento del equipo a lo largo del tiempo. Estos gráficos pueden incluir métricas como el uso de CPU, la cantidad de memoria RAM utilizada, el tráfico de red, etc.
    3. Alertas: Observium puede generar alertas en tiempo real si se detecta un problema con el equipo. Las alertas pueden incluir mensajes de correo electrónico o mensajes de texto.
    4. Reportes de rendimiento: Observium generará reportes detallados que incluyen información sobre el rendimiento del equipo a lo largo del tiempo. Estos reportes pueden ayudarle a detectar tendencias y problemas a largo plazo en el equipo.
    5. Estadísticas: Observium generará estadísticas para ayudarle a entender mejor el rendimiento del equipo. Por ejemplo, puede obtener estadísticas sobre el uso de la CPU, la memoria RAM, el tráfico de red, etc.

Graficos

Observium genera gráficos de rendimiento para ayudar a visualizar cómo se está desempeñando un dispositivo o sistema. Estos gráficos pueden incluir métricas como uso de CPU, uso de memoria RAM, tráfico de red, entre otros. Algunos de los tipos de gráficos de rendimiento que puede ver en Observium incluyen:

    1. Gráficos de línea: estos gráficos muestran cómo se desempeña una métrica a lo largo del tiempo. Por ejemplo, puede ver cómo ha cambiado el uso de CPU en las últimas 24 horas.
    2. Gráficos de barras: estos gráficos muestran el valor de una métrica en un momento determinado. Por ejemplo, puede ver cuánta memoria RAM se está utilizando en este momento.
    3. Gráficos de áreas: estos gráficos muestran cómo se están utilizando varias métricas al mismo tiempo. Por ejemplo, puede ver cómo se están utilizando la CPU y la memoria RAM al mismo tiempo.
    4. Gráficos de histograma: estos gráficos muestran la distribución de una métrica a lo largo del tiempo. Por ejemplo, puede ver cuánto tiempo se ha utilizado una determinada cantidad de CPU.

Los gráficos de rendimiento en Observium son interactivos y se pueden personalizar. Por ejemplo, puede elegir qué métricas se muestran, qué período de tiempo se cubre y cómo se representan los datos. Esto le permite obtener una visión detallada del rendimiento de sus dispositivos y sistemas y tomar decisiones informadas sobre cómo optimizarlos.

Alertas

Observium permite configurar alertas para notificar cuando se detectan problemas en los dispositivos y sistemas que se están monitoreando. Estas alertas pueden enviarse por correo electrónico, mensajes de texto, notificaciones push, entre otros medios. Algunos ejemplos de cuándo se pueden activar las alertas incluyen:

    1. Uso de CPU excesivo: si un dispositivo está utilizando demasiada CPU, puede recibir una alerta para investigar la causa del problema.
    2. Baja disponibilidad de un dispositivo: si un dispositivo deja de responder o no está disponible, puede recibir una alerta para tomar acción.
    3. Tráfico de red anormal: si el tráfico de red en un dispositivo supera los límites previamente establecidos, puede recibir una alerta para investigar si hay un problema de seguridad o una posible amenaza.
    4. Uso de memoria RAM excesivo: si un dispositivo está utilizando demasiada memoria RAM, puede recibir una alerta para investigar la causa del problema.

Reportes de Rendimiento

Observium ofrece la capacidad de generar reportes de rendimiento para ayudar a los usuarios a comprender mejor cómo están funcionando sus dispositivos y sistemas. Estos reportes pueden incluir información sobre el uso de la CPU, la memoria RAM, el tráfico de red, el almacenamiento, entre otros aspectos. Algunos ejemplos de reportes de rendimiento incluyen:

    1. Uso de la CPU: un reporte que muestra el uso de la CPU en un dispositivo o sistema a lo largo del tiempo.
    2. Tráfico de red: un reporte que muestra el tráfico de red en un dispositivo o sistema, incluyendo información sobre el tráfico entrante y saliente.
    3. Uso de la memoria RAM: un reporte que muestra el uso de la memoria RAM en un dispositivo o sistema a lo largo del tiempo.
    4. Almacenamiento: un reporte que muestra el uso de almacenamiento en un dispositivo o sistema, incluyendo información sobre el espacio disponible y utilizado.

Estos reportes pueden ser útiles para identificar tendencias y problemas a largo plazo, y para tomar decisiones informadas sobre cómo optimizar el rendimiento de los dispositivos y sistemas. Además, estos reportes pueden exportarse en diferentes formatos, como PDF o CSV, para compartirlos con otros miembros del equipo o con los departamentos de TI.
